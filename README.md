# falkadeh mohsen pasha 1099
آینده در دستان شماست — فالکدهٔ محسن پاشا مبتنی بر ترکیبی از هوش مصنوعی، منطق انسانی، دانش باستانی و علم نجوم.


## تغییرات مهم (تازه اضافه‌شده)
- یک کلاینت استاتیک جدید اضافه شد: `index.html` (نمای اصلیِ وب‌سایت).
- یک proxy امن سمت سرور اضافه شد: `api/anthropic.js` (Serverless function) — این فانکشن درخواست‌های کلاینت را به Anthropiс فوروارد می‌کند و از قرار گرفتن کلید API در سمت کاربر جلوگیری می‌کند.

> توجه: فایل‌های بالا مستقیماً روی شاخهٔ `main` اضافه شده‌اند.


## راه‌اندازی و انتشار (پیشنهاد شده: Vercel)
این پروژه برای انتشار آسان روی Vercel آماده شده است (پشتیبانی از صفحات استاتیک و Serverless functions در مسیر `api/`). نکات لازم:

1. ایجاد پروژه در Vercel
   - وارد حساب Vercel خود شوید و مخزن `Drmohsenkhoshbahar788-sys/falkadeh1099` را import کنید.
   - Vercel به‌طور خودکار فایل `index.html` را به‌عنوان صفحهٔ استاتیک و فایل‌های داخل `api/` را به‌عنوان توابع سرورلس شناسایی می‌کند.

2. اضافه کردن متغیر محیطی
   - در تنظیمات پروژهٔ Vercel → Environment Variables، یک متغیر به نام `ANTHROPIC_API_KEY` اضافه کنید و مقدار کلید API Anthropic را قرار دهید.
   - Scope را برای Production (و در صورت نیاز Preview) تنظیم کنید.

3. تنظیم دامنهٔ سفارشی
   - در Vercel → Domains دامنهٔ سفارشی خود را اضافه کنید (مثلاً: `falkadehmohsenpasha1099.com`).
   - مطابق دستورالعمل Vercel، رکوردهای DNS را در پنل دامنه تنظیم کنید (CNAME یا A/ALIAS بسته به نوع دامنه).
   - پس از پراپگیشن DNS، Vercel سایت را منتشر و HTTPS را خودکار فعال می‌کند.

4. تست سریع پس از انتشار
   - فراخوانی ساده (غیر streaming):
     curl -X POST "https://<YOUR_DOMAIN>/api/anthropic" \
       -H "Content-Type: application/json" \
       -d '{"model":"claude-sonnet-4-6","max_tokens":100,"system":"سلام","messages":[{"role":"user","content":"سلام"}],"stream":false}'

   - از آنجا که کلاینت موجود در `index.html` از `fetch('/api/anthropic', ...)` استفاده می‌کند، پس از deploy باید همه چیز بدون نیاز به تغییر آدرس کار کند.

- هرگز کلید API را در کد سمت کلاینت یا در README عمومی قرار ندهید. همیشه از متغیرهای محیطی سرور (Server-side env vars / Secrets) استفاده کنید.

  - اعتبارسنجی ورودی‌ها (طول prompt، کاراکترهای نامطلوب) تا جلوی ارسال payloadهای بسیار بزرگ یا مخرب گرفته شود.
  - لاگ‌برداری خطاها با حذف داده‌های حساس (هرگز مقد��ر کلید یا متن حساس را لاگ نکنید).


## فایل‌های کلیدی در این مخزن
- `index.html` — نسخهٔ استاتیکِ رابط کاربری (کلاینت) که از `/api/anthropic` فراخوانی می‌کند.
- `api/anthropic.js` — Serverless proxy که درخواست‌ها را به Anthropic فوروارد می‌کند؛ مقدار کلید را از env `ANTHROPIC_API_KEY` می‌خواند.
- سایر فایل‌ها و دموها (اگر وجود دارد) در ریشه یا پوشه‌های مرتبط.


## اجرای محلی (تست سریع)
- با Vercel CLI (توصیه):
  1. نصب: `npm i -g vercel`
  2. در شاخهٔ پروژه: `vercel login` و سپس `vercel dev`
  3. قبل از اجرای `vercel dev` مقدار `ANTHROPIC_API_KEY` را در `.env` محلی یا به‌صورت متغیر محیطی قرار دهید (فقط برای تست محلی؛ `.env` را به گیت کامیت نکنید).

- یا برای تست بدون سرورلس، می‌توانید proxy را با یک اسکریپت کوچک Node.js/Express محلی اجرا و محیط را شبیه‌سازی کنید (در صورت نیاز راهنمایی می‌دهم).
